hexo-部署

发表于 2023-06-06

这篇文章主要怕自己太久没用 hexo ,不记得部署了

在项目用git bash

先hexo g 生成静态文件

hexo s

hexo d 

 部署到github 仓库,注意密钥一个月生成一次,用经典token 选一个repo 即可

因为部署可能会延迟,所有想看网页效果可以先 hexo s 在localhost先查看

注意:
hexo d可能会延迟,不要急,多等一会

写文章: hexo new “post title with whitespace” (.md就不用了)

hexo会自动在source/.post/里生成新文章,配置啥的都整好了

如果出错了大不了从头再来:从全局安装hexo框架:

npm install -g hexo-cli

开始 ( 注意:如果之前安装过了记得去删掉)
仓库如果用ssh的,记得配一个ssh key 详情看: https://blog.csdn.net/intelrain/article/details/82733876

主题 :找到正确的官方文档很重要!!!
